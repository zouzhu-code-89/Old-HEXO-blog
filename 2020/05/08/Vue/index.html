Vue 响应式框架教程

2020-05-08

Vue.js (读音 /vju:，类似于 view) 是一套构建用户界面的渐进式框架，与其他重量级框架不同的是，Vue 采用自底向上增量开发的设计，Vue 的核心只关注视图层，并且非常容易学习，非常容易与其他库或已有项目整合，另一方面 Vue 完全有能力驱动采用(单文件组件)和(Vue生态系统支持的库)开发复杂单页应用. Vue.js 的目标是通过尽可能简单的 API 实现响应的数据绑定和组合的视图组件。


第 2 章 Vue 生命周期

我们知道 vue 是一个构建数据驱动的 web 界面的渐进式框架，那么 vue 生命周期是什么 ?
vue 生命周期是指 vue 实例对象从创建之初到销毁的过程，vue 所有功能的实现都是围绕其生命周期进行的，在生命周期的不同阶段调用对应的钩子函数实现组件数据管理和DOM渲染两大重要功能，我们来看官网给的 vue 生命周期图片。

Vue 选项 / 生命周期

3.2 optionMergeStrategies

类型：{ [key: string]: Function }

自定义合并策略的选项。

合并策略选项分别接收在父实例和子实例上定义的该选项的值作为第一个和第二个参数，Vue 实例上下文被作为第三个参数传入。

3.3 devtools

类型：boolean 默认值：true (生产版为 false)

配置是否允许 vue-devtools 检查代码。开发版本默认为 true，生产版本默认为 false。生产版本设为 true 可以启用检查。

3.4 errorHandler

类型：Function 默认值：undefined

指定组件的渲染和观察期间未捕获错误的处理函数。这个处理函数被调用时，可获取错误信息和 Vue 实例。

3.5 warnHandler

类型：Function 默认值：undefined

为 Vue 的运行时警告赋予一个自定义处理函数。注意这只会在开发者环境下生效，在生产环境下它会被忽略。

3.6 ignoredElements

类型：Array<string | RegExp> 默认值：[]

须使 Vue 忽略在 Vue 之外的自定义元素 (e.g. 使用了 Web Components APIs)。否则，它会假设你忘记注册全局组件或者拼错了组件名称，从而抛出一个关于 Unknown custom element 的警告。

3.7 keyCodes

类型：{ [key: string]: number | Array<number> } 默认值：{}

给 v-on 自定义键位别名。

3.8 performance

类型：boolean 默认值：false (自 2.2.3 起)

设置为 true 以在浏览器开发工具的性能/时间线面板中启用对组件初始化、编译、渲染和打补丁的性能追踪。只适用于开发模式和支持 performance.mark API 的浏览器上。

3.9 productionTip

类型：boolean 默认值：true

用法： 设置为 false 以阻止 vue 在启动时生成生产提示。

Vue 数据储存与传递

数据在应用架构是非常重要的，我们需要将数据存储起来，有时候我们也要传递数据，下面则是 Vue 数据处理的方案

5.1 响应数据 data

data 是 Vue 中储存数据的属性，且数据具有响应式

1.1 Data 为什么具有响应式 *：Vue 会将 Date 里的属性数据转换为 *Getter 和 Setter，通过 get 和 set 操作，从而让 data 的属性能够响应数据的变化

1.2 Data 数据 ：一旦观察期过后，你就无法在 data 数据对象上添加响应式属性了，因此推荐在创建实例之前，就声明好属性

1.3 Data 组件 ：需要将 Data 定义成一个函数并返回一个对象，因为组件会被多次引用，如果单纯是一个对象 {}，数据是共享

1.4 Data 数据深拷贝 ： 通过将 vm.$data 传入 JSON.parse(JSON.stringify(…)) 得到深拷贝的原始数据对象

除了上面的方式访问数据以外，我们还可以通过 $data 访问 ，它是对实例数据对象的引用

5.2 组件数据 props

组件上也可以传递数据，我们使用组件时 MyComponent，也可以携带一些数据给实例

1.1 Props 类型是 Array | Object

1.2 Props 数据是不能修改的，因为数据是单向传递的，但是我们可以在 Data 中，将 Props 里的数据进行绑定，然后维护 Data

1.3 如果你要传递数字，布尔值，数组，对象，需要使用 v-bind ，不然传递的仅仅是字符串

使用 Object 类型，type 数据类型，default 默认值，required 是否是必传参数

你可以基于对象的语法使用以下选项：

1.1 type ：String、Number、Boolean、Array、Object、Date、Function、Symbol

1.4 validator: Function 自定义验证函数会将该 prop 的值作为唯一的参数代入

$props 代理起了 props 对象，我们在外部可以这样访问

我们还可以通过 propsData 给 props 传递参数，只用于在 new vue 创建的实例中，主要用于向 props 进行数据传递

5.4 计算属性 computed

1.1 计算属性可以在 Computed 中返回的数据，混入到 Vue 实例中

1.2 计算属性的结果会被缓存，除非依赖的响应式属性变化，才会重新计算，非响应式数据，则不会更新

类型 { key: function | { get: function, set: function} }

get / set 类型是什么样的 ？ 设置新值时调用 set ，获取值时调用 get

5.5 实例方法 methods

用于在实例中定义方法，我们可以通过实例调用

5.6 监听 Data watch

watch 用于监听 data 中的数据变化，当 data 数据变化时，会触发 watch 中的属性事件

1.1 immediate: true 数据绑定的时候，就执行 watch，默认是在变化时，才执行 watch

1.2 deep:true 当 data 是对象时，则不能监听，设置这个后，就可以对象里的每一个属性进行监听

1.3 handler 表示监听的函数

可以在数据发生变化的时候，调用 methods 中的方法

$watch 除了在实例中调用声明监听以外，我们还可以在外部声明

5.7 数据传递 provide / inject

*provide and inject *

主要用于不同组件间的数据传递，主要在开发高阶插件 / 组件库时使用，并不推荐用于普通应用程序代码中，它的绑定并不是响应的，然而，如果你传入了一个可监听的对象，那么其对象的属性还是可响应的

模板正常显示数据，测试成功

Vue 创建模板

Vue 是面向组件开发，所以也框架也提供了几种创建模板的方法，render > template > el

6.1 挂载 el

当实例中没有定义 template 属性，或者 render，默认将 el 挂载的 dom 转化成 template 并显示

6.2 template

可以在实例中直接定义 template

也可以在外部定义 tempalte

6.3 render

render 是通过 JavaScript 创建 html，它允许我们做出更多的判断，参数类型 createElement，返回类型 createElement

createElement : 创建 DOM，第一个参数是 dom<string，component>，第二个参数是 Object，参数三 String | Array[element]

将组件传递给第一个参数

Vue 里面的 createElement 函数，这个函数的作用就是生成 VNode 节点，rander 得到这个 VNode 节点之后，返回给 vue.js 的 mount 函数，渲染成真实的 Dom 节点，并挂载到根节点上.

6.4 renderError

render 函数遭遇错误时，提供另外一种渲染输出，它的作用是当 render 出错时，提供错误发生时的解决方案，只在开发模式下使用，其错误将会作为第二个参数传递到 renderError，这个功能配合 hot-reload 非常实用

Vue 对象组合

Vue 可以实现继承，形成父子关系，或者混入

7.1 parent

parent 是指定父实例，两者之间建立父子关系

$parent and $children

1.1 子实例访问父实例数据 ：因为我们指定了一个父实例，所以我们可以通过实例的 $parent 方法访问父实例的数据

1.2 父实例访问子实例数据 ：所有的子实例，将封装到 children 数组中，我们可以通过实例的 $children 方法访问子数据

注意 ：我们应该节制地使用 $parent 和 $children，它们的主要目的是作为访问组件的应急方法，更推荐用 props 和 events 实现父子组件的通信 。

7.2 mixins

混入对象可以像正常的实例对象一样包含，指令，过滤器，等等选项，这些选项将会被合并到最终的实例选项中，如果你的混入包含一个 created 钩子，而创建实例本身也有一个 created，那么两个函数都会被调用，Mixin 钩子按照传入顺序依次调用，并在调用实例组件自身的钩子之前被调用，也就是先调用混入

mixins 类型是 Array(Object)

7.3 extends

extends 和 mixins 类似，通过暴露一个 extends 对象到组件中使用，执行顺序：extends > mixins > new，extends 只能暴露一个 extends 对象，暴露多个 extends 不会执行

Vue 实例属性资源

vue 选项资源，资源包括，指令，过滤器，组件，都是 vue 内部实例的资源，只能在实例挂载元素范围内使用

8.1 指令 directives

directives 是在实例内部定义的指令，参数类型是 { key : {key: function} } ，生命周期跟全局是一样的.

  • bind : 只调用一次, 指令第一次绑定到元素时调用, 可以定义一个初始化动作
  • inserted : 指令绑定的标签插入到 document 中时
  • update ：指令绑定的标签内数据发生变化时, 触发
  • componentUpdated ：被绑定元素所在模板完成一次更新周期时调用
  • unbind : 只有在你的 dom 元素被 VUE 移除（你手动删除也不可以）时才会被调用

8.2 过滤器 filters

filters 是在实例内部定义的过滤器，参数类型是 { key : function }，只能作用于挂载元素内部

8.3 组件 components

components 实在实例内部定义的组件，参数类型是 {key: Object}，只能作用于挂载元素内部

Vue 获取生命周期

用于操作组件的生命周期，下面实例的方法就是关于组件的生命周期

13.1 vm.$mount

如果 Vue 实例在实例化时没有收到 el 选项，则它处于未挂载状态，没有关联的 DOM 元素，可以使用 vm.$mount() 手动地挂载一个未挂载的实例

13.2 vm.$forceUpdate

迫使 Vue 实例重新渲染。注意它仅仅影响实例本身和插入插槽内容的子组件，而不是所有子组件

13.3 vm.$nextTick

修改数据之后立即使用它，浏览器此时 DOM 已经更新了

13.4 vm.$destory

完全销毁一个实例。清理它与其它实例的连接，解绑它的全部指令及事件监听器。

触发 beforeDestroy 和 destroyed的钩子。

在大多数场景中你不应该调用这个方法。最好使用 v-if 和 v-for 指令以数据驱动的方式控制子组件的生命周期。

Vue 特殊属性

特殊属性，作用在标签上，主要用于做标记，或使用一种特殊的算法，下面我们将挨个介绍

14.1 key

官网上说 ：如果不使用 key，移动元素的话，vue 会使用一种最大限度减少动态元素，并且尽可能的尝试 就地修改和复用相同类型元素的算法，而使用 key 时，它会基于 key 的变化 重新排列元素顺序，并且会移除 key 不存在的元素

我们要根据实际情况使用 key ，因为不使用 key 会增强性能

14.2 ref

Vue 指令

指令也是框架中的一部分，上面我们已经体验了自定义指令，下面的这些是框架为我们封装的内置指令

15.1 v-text

参数类型为 String，作用是更新元素的 textContent，它不会渲染 Dom 标签，而是按字符串输出，它是没有闪烁问题的.

15.2 v-html

它的作用是将 data 中的数据按超文本解析的方式进行解析，这是 和 v-text 做不到的一点

15.3 v-show

它的作用跟 v-if 的作用一样，但是它是将 DOM 元素的样式，设置成 display ：none；而 if 则是删除 DOM

15.4 v-if / v-else-if / v-else

这里就不多做介绍了，学过语言的是个猪都知道，但是还是要说明的是，它是真实删除 DOM 的
